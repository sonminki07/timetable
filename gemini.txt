# 가톨릭대 시간표 분석기 프로젝트 문서 (Gemini CLI Brain)

이 문서는 프로젝트의 초기 상태부터 현재(React/Next.js 버전)까지의 모든 핵심 로직과 개발 이력을 담고 있습니다.

## 1. 프로젝트 개요
- **이름**: 가톨릭대 시간표 분석기 (Timetable App)
- **원본**: `timetable.html` (V8 Premium 단일 파일 버전)
- **현재 버전**: Next.js 15+ (TypeScript), Zustand, Tailwind CSS 기반의 React 웹 애플리케이션
- **배포**: GitHub (sonminki07/timetable) + Vercel (자동 배포)

## 2. 핵심 로직 (The Engine)

### 2.1. 텍스트 파서 (`lib/utils/parser.ts`)
- 에브리타임 장바구니 목록을 복사+붙여넣기 했을 때, 정규표현식(`Regex`)을 사용하여 과목명, 교수명, 요일, 교시, 강의실 정보를 정확히 추출합니다.
- `timesOnly`, `roomsOnly`, `allTimes` (예: "월1", "월2") 배열을 생성하여 중복 체크의 기반을 마련합니다.

### 2.2. 시간표 생성기 (`lib/utils/generator.ts`)
- **조합 알고리즘**: 모든 그룹에서 과목을 하나씩 선택하여 겹치지 않는 모든 경우의 수를 재귀 함수(`combine`)로 탐색합니다.
- **성능 최적화**: 
  - 렌더링 부하를 줄이기 위해 결과 개수를 상위 50개로 제한합니다.
  - 렌더링 시점이 아닌 생성 시점에 **이동 시간(Distance)**을 미리 계산(`Pre-calculate`)하여 데이터에 포함합니다.
- **필터링**: 필수 공강 요일, 특정 교시 제외, 특정 강의 제외, 연강 정책(파괴 옵션 시)을 실시간 반영합니다.

### 2.3. 추천도 계산기 (`lib/utils/calculator.ts`)
- **기본 점수**: 100점에서 시작.
- **가점 요인**: 선호 요일 공강(+20), 일반 공강(+15), 점심시간 확보(+15), 선호 과목 포함(+15).
- **순위 기반 가점**: 각 그룹 내 강의 순서(1~5순위)에 따라 사용자가 설정한 점수(예: 1순위 +100점)를 부여합니다. 기존 교수명 매칭 방식을 대체했습니다.
- **감점 요인**: 1교시 수업(-30), 연강 제한 초과(설정에 따라 감점 또는 제외).
- **거리 행렬**: 가톨릭대 성심교정의 건물 간 이동 시간을 매트릭스 형태로 정의하여 이동 편의성을 분석합니다.

## 3. 상태 관리 (`store/useTimetableStore.ts`)
- **Zustand + Persist**: 사용자의 모든 입력값과 설정(`Settings`)을 브라우저의 `localStorage`에 자동 저장합니다.
- **비휘발성 관리**: `isGenerating` (생성 중 상태) 등은 저장 대상에서 제외하여 페이지 새로고침 시 UI가 멈추는 현상을 방지했습니다.
- **에러 핸들링**: 생성 중 발생하는 예외를 `try...catch`로 보호하여 서비스 안정성을 높였습니다.

## 4. UI/UX 디자인 (Parity with Original)
- **HTML 완벽 복제**: 원본 HTML의 CSS 변수, 색상표, 다크 모드 테마를 `app/globals.css`에 그대로 이식했습니다.
- **애니메이션**: 사이드바가 나타날 때 `SlideIn` 애니메이션을 적용하고, 조건부 렌더링을 통해 가로 스크롤 레이아웃 꼬임을 원천 차단했습니다.
- **인터랙션**: 1시간(1칸)짜리 강의 블록에만 호버 효과(`can-hover`)를 적용하여 시각적 복잡도를 줄였습니다.
- **드래그 앤 드롭 (`dnd-kit`)**: 그룹 내 강의 순서를 마우스로 끌어서 변경할 수 있으며, 이 순서가 곧 우선순위(1~5순위)가 됩니다.

## 5. 작업 히스토리 (History)
1. **디자인 동기화**: 원본 HTML의 버튼, 카드, 테마 스타일을 리액트에 완벽 반영.
2. **기능 강화**: 특정 강의 제외 버튼(-/+) 추가 및 필터 경고(그룹 파괴) 메시지 구현.
3. **성능 개선**: 렌더링 렉(Lag)을 유발하던 중복 거리 계산 로직을 생성 엔진으로 이전.
4. **레이아웃 수정**: 사이드바 호출 시 페이지가 오른쪽으로 밀리는 가로 스크롤 이슈 해결.
5. **연강 정책 추가**: 연강 초과 시 '감점'과 '조합 파괴' 중 사용자가 선택할 수 있는 옵션 추가.
6. **배포**: 깃허브 원격 저장소 연결 및 Vercel 실시간 호스팅 환경 구축.
7. **순위 시스템 도입**: 교수명 매칭 대신, 각 그룹의 강의 순서(1~5순위)에 따라 가점을 부여하는 직관적인 시스템으로 변경. 드래그 앤 드롭으로 순위 변경 가능.

## 6. 유지보수 가이드
- **디자인 수정**: `app/globals.css`를 수정하십시오.
- **로직 수정**: `lib/utils/` 폴더 내의 각 유틸리티 파일을 수정하십시오.
- **배포 업데이트**: 터미널에서 `git push`를 실행하면 Vercel이 자동으로 웹사이트를 업데이트합니다.
